## Findings

### [P1 — FIXED] `src/poly/telegram-commands.ts` — `/poly` handler missing auth guard

**Commit:** `d186090`

The `bot.command('poly', ...)` handler in `telegram-commands.ts` had no `isAuthorised` check.
grammy command handlers run independently of `bot.on('message:text')` middleware — any Telegram user who knew the bot's username could call `/poly halt`, `/poly resume`, and read all internal state.

**Fix:** Added `if (!ALLOWED_CHAT_ID || ctx.chat?.id.toString() !== ALLOWED_CHAT_ID) return;` as the first line of the handler. Imported `ALLOWED_CHAT_ID` from config.

### [P1 — FIXED] `src/poly/telegram-commands.ts:291` — `renderPnl` unrealized sum unfiltered

**Commit:** `d186090`

`SELECT COALESCE(SUM(unrealized_pnl), 0) FROM poly_positions` had no join against open trades.
A stale `poly_positions` row left by a crash mid-resolution would silently inflate the unrealized total.

**Fix:** Changed to `INNER JOIN poly_paper_trades t ON t.id = p.paper_trade_id WHERE t.status = 'open'`, consistent with how `strategy-engine.ts` queries the same data.

**Test added:** `telegram-commands.test.ts` — "excludes stale poly_positions rows for closed trades from unrealized total". Verified RED before fix, GREEN after.
